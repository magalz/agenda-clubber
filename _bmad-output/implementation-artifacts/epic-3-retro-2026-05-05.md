## 2. Stories — Highlights e Achados

### 3.1 — Grid do Calendário e Visualização de Saúde da Cena
- **Wins:** Backend-first com stub validado; contratos `ConflictLevel`, `HealthPulseMap` estabilizados para todo o épico; 179 testes passando; estética Line-over-Black implementada.
- **Achados:** 17 findings no review adversarial, 7 corrigidos. Timezone mismatch (UTC vs SP) — `formatDateKey` com `Intl.DateTimeFormat`. `.limit(1)` sem ORDER BY corrigido. `new Date()` como default param banido.
- **Deferred:** Events table (3.2), Algorithm (3.3), Zustand/TanStack (3.2).

### 3.2 — Cadastro de Evento e Geolocalização
- **Wins:** Migration 008 (18 colunas events), Server Actions `createEvent`/`updateEvent` com pipeline completo (Zod → geocode → timezone → UTC → persist); Zustand + TanStack Query introduzidos; Realtime via Supabase; 208 testes.
- **Achados:** Zod v4 breaking changes cascatearam para artists/auth (`.errors`→`.issues`, `errorMap`→`message`). `tz-lookup` ESM-only incompatível — pivot para Open-Meteo. 6 commits pós-merge de hardening E2E. `shadcn Select` `onValueChange` retorna `string | null`.
- **Deferred:** DEBT-3.2-A (Bio do Test DJ flake CI), cálculo de conflito (3.3), RLS (3.4).

### 3.3 — Motor de Regras Algoritmo v1.2
- **Wins:** 3 famílias de regras puras + orquestrador `server-only`; compute-on-write com recompute transacional de vizinhos; justificativas em PT-BR canônicas; 327 testes; Health Pulse real substituindo stub.
- **Achados:** Decisão Q5 (compute-on-write) validada — reads triviais, writes mais pesados mas bounded. Performance worst-case ~3s (acima de NFR1 <2s) — otimização de batch lookup de artistas aplicada.
- **Deferred:** 5 tópicos para decisão de produto registrados no story file. Tech debt: lineup ↔ artists FK (matching string-normalizado).

### 3.4 — Privacidade Granular e Status do Evento
- **Wins:** `filterEventForViewer` puro seguindo padrão `artists/visibility.ts`; toggles de privacidade; `updateEventStatus` com FR16 reveal; query cross-collective; RLS migration 010; 356/358 testes.
- **Achados:** Review adversarial: 5 patches aplicados (O(N²) useMemo, type safety, guarda array vazio, no-op guard, IF NOT EXISTS RLS). 1 divergência estrutural: RLS bloqueia rows com todas flags `false`, mas app-layer quer mostrar `genrePrimary`. 4 deferrals (pre-existing).
- **Deferred:** Health Pulse cross-collective, Realtime cross-collective.

### 3.5 — Componente de Delay Ético para Conflitos Críticos
- **Wins:** `EthicalDelayButton` reutilizável com Dialog + Progress + countdown 3s; toast educativa; integração condicional no DayDetailSheet; 371 testes.
- **Achados:** `vi.useFakeTimers()` congela `setTimeout` do React — corrigido com `{ toFake: ['setInterval', 'clearInterval'] }`. Merge sem review formal — sem bugs detectados mas sem verificação adversarial.
- **Deferred:** Nenhum.

## 10. Lições-Chave

1. **Backend-first com stub** destrava UX paralela e mantém testes determinísticos — padrão ouro do projeto.
2. **Defesa em profundidade** (RLS + app-layer) consolidado como padrão para tudo com `privacy_settings`.
3. **Review adversarial 3 camadas** captura issues de segurança/integridade antes de `main` — tornar obrigatório, sem exceções.
4. **Migrations manuais** são fricção aceita, mas CI e prod precisam usar o **mesmo** mecanismo de migração.
5. **Tracking centralizado de débito** é pré-condição para visibilidade — `deferred-work.md` com 1 entrada não escala. GitHub Issues é o padrão da indústria.
6. **Seed determinístico** é ESSENCIAL para E2E — `ON CONFLICT DO UPDATE` mascara bugs. Usar `DELETE` + `INSERT`.
7. **Agente de QA dedicado** no ciclo de story previne o padrão de "6 commits de hardening pós-merge".

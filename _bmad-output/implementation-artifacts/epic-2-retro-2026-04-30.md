## 2. Stories — Highlights e Achados

### 2.1 — Criação on-the-fly + Notificação
- **Wins:** backend-first (sem UI), Server Action + QStash stub, schema migration nullable preservando integridade.
- **Achados:** drizzle-kit forçou migration manual; QSTASH_TOKEN vazio falhava silenciosamente — guard adicionado.
- **Deferred:** e-mail real, idempotência do webhook, CHECK constraint DB.

### 2.2 — Busca Global Cmd+K
- **Wins:** Command Palette global, `SearchHit` discriminated union, `ArtistIdentityCard` com variants.
- **Achados:** cmdk com `shouldFilter=true` (default) ocultava resultados — `<Command shouldFilter={false}>` é obrigatório no nosso caso. Wildcard injection em `LIKE` corrigido com `replace(/[%_]/g, '\\$&')`.
- **Deferred:** índice `pg_trgm`, otimização `next/image`.

### 2.3 — Busca Obrigatória + Claim + Privacy
- **Wins:** Migration 005 (status/bio/privacy_settings), RLS atualizado, máquina de 3 steps no onboarding, 105 testes.
- **Achados:** 7 iterações de CI até E2E verde (secrets invertidas, drizzle-kit lock, race condition `rowCount ?? 1`). Validação de magic bytes sem try/catch causava 500. Upload antes de validar `profileId` exclusivo deixava arquivos órfãos.
- **Deferred:** UI granular de privacy por campo, enum admin role (Story 5.1).

### 2.4 — Perfil Público + SEO
- **Wins:** Migration 006 (slug), `slugify` + `uniqueSlug`, `filterArtistForViewer` puro, `generateMetadata` dinâmico, 138 testes.
- **Achados:** slugify retornava `''` para nomes não-Latinos (colidia no UNIQUE) — fallback `'artist'` + sufixo. 404 vazava existência de Ghost — alinhado com soft-404 do `cacheComponents`. Dupla query (metadata + page) sem cache compartilhado — `React.cache()` aplicado.
- **Deferred:** cobertura E2E de `rejected`, benchmark de cache.

## 3. Padrões Recorrentes (3 stories ou mais)

1. **Wildcard injection em `LIKE`/`ilike`** — Stories 2.2 e 2.3. Mesmo fix aplicado com regex levemente diferente em cada callsite (`[%_]` vs `[%_\\]` vs `[\\%_]`). **Action item:** extrair helper `escapeLikePattern()` antes do Épico 3.
2. **Error silencing** — Stories 2.1 (QStash token), 2.2 (`res.error`), 2.4 (`validateMagicBytes`). Padrão consolidado: guard explícito + log em vez de fail-forward silencioso.
3. **Race conditions com `rowCount ?? 1`** — Story 2.3. Padrão estabelecido: usar `?? 0` (conservative) em ops idempotentes.

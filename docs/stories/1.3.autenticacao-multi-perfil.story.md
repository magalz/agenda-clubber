## Story
**As a** User (Collective or Artist),
**I want** to create an account and select my role,
**so that** I can access the specific features and data relevant to my profile in the ecosystem.

## Acceptance Criteria
1. [x] User can sign up/sign in via Supabase Auth (Email/Password) in `apps/web`. [Source: architecture.md#3]
2. [x] Mandatory role selection ('collective' or 'artist') upon first login if profile is incomplete. [Source: prd.md#FR1]
3. [x] Successful creation of a `profiles` record linked to `auth.users` with the selected role. [Source: architecture.md#3]
4. [x] Automatic creation of a child record in `collectives` or `artists` table depending on the role. [Source: supabase/migrations/..._initial_schema.sql]
5. [x] Middleware/Route protection in `apps/web` to ensure users only access their role-specific areas.
6. [x] User's `username` must be unique and validated in the `profiles` table.

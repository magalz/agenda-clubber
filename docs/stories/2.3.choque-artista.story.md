# Story 2.3: Motor de Conflito: Choque de Artista

> **Status:** Ready for Review
> **Versão:** 1.0
> **Responsável:** River (SM)

## Story
**As a** Collective,
**I want** to be alerted if an artist I am adding to an event is already booked for another event at the same time,
**so that** I can coordinate with the other collective or find a replacement before announcing.

## Acceptance Criteria
1. [x] Detection logic must identify if an artist (by `artist_id`) has overlapping `start_time` and `end_time` across different events. [Source: prd.md#FR4]
2. [x] Alerts should be triggered during event creation or update when adding artists.
3. [x] The system must allow the conflict to exist (soft-blocking) but must display a clear visual warning (e.g., Red Alert) in the dashboard.
4. [x] Integration with the `event_artists` junction table to track bookings. [Source: supabase/migrations/..._initial_schema.sql]
5. [x] Conflict alerts must be visible even if the conflicting event is 'Anonymous' (it should show "Conflict with Anonymous Event" without leaking the other collective's name or event title).

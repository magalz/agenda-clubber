# Story 1.4: Gestão Básica de Perfil (Avatar/Bio)

> **Status:** Draft
> **Versão:** 1.0
> **Responsável:** River (SM)

## Story
**As a** User (Collective or Artist),
**I want** to update my basic profile information such as Bio and Avatar,
**so that** I can present myself correctly to the community.

## Acceptance Criteria
1. [ ] User can view their current profile data (Bio, Avatar, Username) in `apps/web`. [Source: architecture.md#3]
2. [ ] Form to update `bio` and `avatar_url` in the `profiles` table. [Source: architecture.md#3]
3. [ ] Ability to upload/update an avatar image to Supabase Storage.
4. [ ] Validation of bio length (max 500 characters).
5. [ ] Changes are reflected in both the `profiles` table and any role-specific table if relevant (e.g., `artists.avatar_url` if redundant, but architecture says `profiles` is the source).
6. [ ] Proper feedback (success/error toast) after profile update.

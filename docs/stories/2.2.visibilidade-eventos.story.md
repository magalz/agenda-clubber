# Story 2.2: Níveis de Visibilidade de Reserva (Anônimo/Identificado)

> **Status:** Done
> **Versão:** 1.0
> **Responsável:** River (SM)

## Story
**As a** Collective,
**I want** to choose between 'Anonymous' or 'Identified' visibility for my 'Idea' status events,
**so that** I can reserve a date in the calendar without necessarily revealing my brand to competitors during early planning.

## Acceptance Criteria
1. [x] Events in 'Idea' status with `visibility = 'Anonymous'` must hide the `collective_id` and `title` from other users in public views. [Source: prd.md#FR3]
2. [x] Events in 'Idea' status with `visibility = 'Identified'` must show the `collective_id` (or brand name) but can still hide specific details if needed (to be refined in UI).
3. [x] RLS policies must correctly restrict the data returned based on the user's relationship to the event. [Source: supabase/migrations/..._initial_schema.sql]
4. [x] UI components must handle 'masked' data for anonymous events (e.g., show "Reserved Slot" instead of the real title).
5. [x] Public Calendar view (to be implemented) must respect these visibility levels.

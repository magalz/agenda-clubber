# Story 3.4: Atualização em Tempo Real (Realtime)

> **Status:** Draft
> **Versão:** 1.0
> **Responsável:** River (SM)

## Story
**As a** Collective,
**I want** to see new events and updates appearing in my calendar instantly without refreshing the page,
**so that** I am always working with the most up-to-date information in the scene.

## Acceptance Criteria
1. [ ] Integration with Supabase Realtime to subscribe to changes in the `events` table. [Source: prd.md#FR4, architecture.md#2]
2. [ ] When an event is created, updated, or deleted by ANY collective, the Calendar Grid must update automatically for all active users.
3. [ ] Real-time updates must respect the 'masking' logic (anonymized data must still be anonymized when pushed via realtime).
4. [ ] Visual feedback (e.g., a brief flash or toast notification) when a new event is added to the scene.
5. [ ] Performance: Real-time updates should not cause significant UI lag or duplicate renders.

- [ ] **Task 2: Realtime Hook/Hookup**

  - [ ] Implement the logic to merge real-time updates into the existing calendar state.
- [ ] **Task 3: Masking in Realtime**
  - [ ] Verify that Supabase Realtime payloads respect RLS or ensure the frontend applies `maskEvent` immediately upon receiving a push.

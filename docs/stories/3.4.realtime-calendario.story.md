# Story 3.4: Atualização em Tempo Real (Realtime)

> **Status:** Done
> **Versão:** 1.0
> **Responsável:** River (SM)

## Story
**As a** Collective,
**I want** to see new events and updates appearing in my calendar instantly without refreshing the page,
**so that** I am always working with the most up-to-date information in the scene.

## Acceptance Criteria
1. [x] Integration with Supabase Realtime to subscribe to changes in the `events` table. [Source: prd.md#FR4, architecture.md#2]
2. [x] When an event is created, updated, or deleted by ANY collective, the Calendar Grid must update automatically for all active users.
3. [x] Real-time updates must respect the 'masking' logic (anonymized data must still be anonymized when pushed via realtime).
4. [x] Visual feedback (e.g., a brief flash or toast notification) when a new event is added to the scene.
5. [x] Performance: Real-time updates should not cause significant UI lag or duplicate renders.

# Story 2.1: CRUD Base de Eventos

> **Status:** Ready for Review
> **Versão:** 1.0
> **Responsável:** River (SM)

## Story
**As a** Collective,
**I want** to create, read, update, and delete events in my dashboard,
**so that** I can maintain my organization's strategic calendar.

## Acceptance Criteria
1. [x] Form to create a new event with: `title`, `start_time`, `end_time`, `status` (Idea, Planning, Confirmed), and `visibility`. [Source: prd.md#FR2]
2. [x] List view of events owned by the current Collective. [Source: architecture.md#3]
3. [x] Ability to edit existing event details.
4. [x] Ability to delete an event (with confirmation).
5. [x] Automatic association of the event with the user's `collective_id`. [Source: supabase/migrations/..._initial_schema.sql]
6. [x] Basic validation: `end_time` must be after `start_time`.
7. [x] UI follows the industrial dark theme standards.

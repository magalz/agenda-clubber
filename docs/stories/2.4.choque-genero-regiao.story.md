# Story 2.4: Motor de Conflito: Choque de Gênero e Região

> **Status:** Done
> **Versão:** 1.0
> **Responsável:** River (SM)

## Story
**As a** Collective,
**I want** to be warned if another event of the same musical genre is happening in the same region on the same date,
**so that** I can evaluate if there is public cannibalization and decide if I should adjust my strategy.

## Acceptance Criteria
1. [x] The system must allow selecting a `genre` (e.g., Techno, House, Trance, Psytrance) for an event. [Source: prd.md#FR4]
2. [x] The `locations` table must support a `neighborhood` or `region` field to identify geographical proximity.
3. [x] Detection logic must identify events that share the same `genre` AND `region` AND have overlapping `start_time`/`end_time`.
4. [x] These warnings should be classified as "Warnings" (Yellow Alert), differentiating them from "Conflicts" (Red Alert - Artist Choque).
5. [x] The alert must show the overlapping event's data, respecting visibility levels (masking if the competitor's event is 'Anonymous').
6. [x] Dashboard UI must display these warnings prominently during event creation/editing.
